# AstrBot 天使之笑 2.0

一个基于 AstrBot 的表情包增强插件，当前版本同时提供两条能力链路：

- 自动监听消息中的图片，使用 LLM 审查后异步偷图
- 在回复阶段支持 `:tag1:tag2:` 多 Tag 组合匹配并插入表情包
- 保留兼容工具 `steal_meme_to_smile`，可按指定分类手动入库本地图片

## 当前主要行为

### 1. 自动偷图

插件会监听消息中的图片：

- 若 LLM 判断图片适合作为表情包，则自动保存
- LLM 会为图片生成 1~4 个标签
- 默认使用第一个标签作为主目录名保存
- 数据使用 SQLite 维护，支持标签索引与使用统计

### 2. 回复渲染

当 LLM 输出类似以下格式时：

- `:开心:`
- `:amused:cat:`
- `:委屈:兔兔:`

插件会在发送阶段尝试匹配对应标签组合，并把文本替换成表情图片。

### 3. 兼容工具入口

工具名：`steal_meme_to_smile`

这个工具用于兼容旧工作流：

- 输入本地图片路径
- 由调用方显式指定 `category`
- 将图片保存到插件目录
- 工具只返回调用结果，不主动向用户发消息

参数：

- `image_path`：本地图片路径，必填
- `category`：必填，明确指定分类
- `description`：可选，分类描述
- `save_name`：可选，保存后的文件名

## 数据目录

插件运行后会在以下目录维护数据：

`data/plugin_data/astrbot_plugin_angel_smile`

常见内容包括：

- `memes.db`：SQLite 数据库，维护表情包、标签、使用次数等信息
- `memes/<tag>/`：表情包图片目录
- `memes_data.json`：旧版数据迁移入口；迁移后可能被备份

## 说明

- 自动偷图和兼容工具入口可以同时存在，前者面向 2.0 主流程，后者面向手动/兼容调用。
- 回复阶段不会清洗 `:tag:` 文本，而是直接在装饰结果阶段解析替换。
- 组合标签越多，匹配越精准；如果没有匹配项，则保持原文本不替换。
